## 검수 시 주의사항 (놓치면 안 되는 것)

> Electron GUI 프로세스는 쉘 환경변수(`LANG`, `LC_ALL`)가 없다. 아래 두 가지를 동시에 확인할 것.
>
> - **tmux 명령** (`list-panes`, `capture-pane`) → `LANG=ko_KR.UTF-8` 필요, 없으면 한글 세션명이 `_`로 치환됨
> - **ps 명령** → `LANG` 적용 **금지**, 적용 시 `lstart` 날짜가 한국어 포맷으로 바뀌어 파싱 오프셋이 깨져 세션 누락됨
> - **PTY 실행** (`open-pty`) → shell args에 `tmux` 풀 경로(`/usr/local/bin/tmux`) 필수, 없으면 `zsh: command not found: tmux` 에러 후 세션 즉시 종료됨
>
> `sessions.js`에서 tmux 명령에는 `TMUX_ENV`, ps/lsof에는 기본 env를 사용하는 것이 정답. `main.js` PTY 코드도 풀 경로 유지.

| 버그 | 심각도 | 관련 파일 |
|------|--------|----------|
| Electron GUI 컨텍스트에서 LANG/LC_ALL 없어 한글 세션명 `_`로 치환 | 수정완료 | sessions.js (LOCALE_ENV 추가) |
| 세션명에 `"` / `'` 포함 시 tmux 명령 shell injection 가능 | 중간 | launcher.js |

## 검수 방법

```bash
# ⚠️ 앱 재시작 — 반드시 인스턴스 수 확인 후 진행
ps aux | grep "Electron \." | grep -v grep | awk '{print $2}' | xargs kill -9 2>/dev/null
sleep 2
# 0 확인 필수
ps aux | grep "Electron \." | grep -v grep | wc -l
cd ~/claude-commander && node_modules/.bin/electron . --remote-debugging-port=9334 > /tmp/cc.log 2>&1 &
sleep 3
# CDP 연결 + 인스턴스 수 동시 확인 (인스턴스 1개여야 함)
curl -s http://localhost:9334/json/version | python3 -c "import json,sys; print('CDP ✅', json.load(sys.stdin).get('Browser',''))"
ps aux | grep "Electron \." | grep -v grep | awk '{print $2, $13}'

# 로그 실시간 확인
tail -f /tmp/cc.log

# tmux 세션 확인
/usr/local/bin/tmux list-panes -a -F "#{session_name}|#{pane_pid}|#{pane_current_path}"

# sessions.js 직접 테스트 (node)
node -e "const {getSessions}=require('./lib/sessions'); getSessions().then(s=>s.forEach(x=>console.log(x.name, x.type, x.status)))"

# CDP로 세션 inject 후 렌더링 검증
# → chrome-devtools MCP evaluate_script 활용
```

> **포트 주의**: 9334는 web-editor 전용. web-editor 실행 중이면 9336 사용 (MCP 연결 안 됨).
> CDP 검증 시 window-shown이 안 트리거된 상태면 sessions를 JS로 직접 inject해서 렌더링 검증 가능.
